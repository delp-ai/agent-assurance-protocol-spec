# AAP Conformance Byte Transcripts

This artifact groups canonical frame vectors into transcript-level gating fixtures.

It is the next step above `09.BYTE.CORPUS.md`: instead of isolated frames, it
defines ordered exchanges that another implementation can replay and compare frame-by-frame.

These transcripts use the same conformance-only convention as the byte corpus:

- post-decryption AAP frame bytes
- `AuthTag` length fixed to `0`
- exact bytes are compared before any transport-specific re-protection

## Transcript Rules

Every byte transcript should publish:

- ordered frame bytes directly or an ordered list of source vectors whose bytes are already
  published canonically in `09.BYTE.CORPUS.md`
- either a frame-local expected parse result or a per-frame semantic outcome when the cited source
  vector already publishes the canonical parse summary
- transcript-level semantic outcome
- any deterministic emitted `WIRE_ERROR`

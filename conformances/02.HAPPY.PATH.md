through final close.

completed successfully and yielded:

- negotiated wire version `1.0`
- `sessionLineageId: lineage.boot_0001`
- initiator `agent.alpha`
- executor `agent.beta`

## Required Validation Checks

- every contract-bound frame repeats the same issuer-scoped contract identity in header and
  payload
- no transition to `Completed` occurs without `STATE phase=completed`
- every phase-changing `STATE` carries `stateTransitionDigest`
- every digest referenced by a required frame resolves to a published canonical object in this
  artifact set or a referenced bootstrap/resume artifact
- `FinalityContext.witnessSetDigest` resolves to a published `FinalityWitnessSet`
- `SETTLE` does not by itself imply semantic success or final close
- `CLOSE` is valid only when the dispute-window condition is satisfied and the finality evidence plus
  close context converge on the same accepted terminal checkpoint set
- `closeTraceDigest` must resolve to the accepted close decision that matches the published
  finality/observation context

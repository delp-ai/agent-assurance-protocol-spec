# AAP Conformance Bootstrap Transcript

This artifact defines a canonical object-form transcript for an unknown-peer join under the
revised `AAP` base profile.

## Participants

- initiator: `agent.alpha`
- responder: `agent.beta`

## Shared Expectations

- both peers support base-profile wire major `1`, minor `0`
- both peers support `aap-base-profile-v1`
- both peers support `aap-predicate-wasm32-v1`
- both peers accept inline handshake-critical manifests
- both peers require protected `HELLO` and `CAPS`
- symbolic manifest refs in this transcript denote either verified pre-pinned bootstrap material or
  manifest objects carried inline within the protected `HELLO` / `CAPS` exchange
- trust-domain attestations, domain-admission objects, and fresh revocation status required for this
  bootstrap are therefore available by protected inline carriage or verified pre-pin before
  contract traffic becomes admissible
- no human override exists at any step

## First Post-Negotiation Frame Check

The next admissible contract-bound frame must carry:

```text
Version: 1.0
```

If it carries `Version: 0.0`, the receiver must reject it as a negotiation-state violation.

## Deterministic Failure Cases Anchored By This Artifact

- `HELLO` received with pre-negotiation version other than `0.0`
- `HELLO` whose `supportedMajorVersions[]` and `supportedMinorRanges[]` lengths differ
- `CAPS` with `chosenMajor` or `chosenMinor` not equal to the deterministic intersection
- `CAPS` that omits required inline manifest material without a valid pre-pinned exception
- `HELLO` or `CAPS` that carries a trust policy digest different from the accepted bootstrap policy
- post-`CAPS` contract-bound frame received before both peers emitted matching chosen values
